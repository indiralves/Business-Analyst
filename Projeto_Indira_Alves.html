SQL · Reporting

1. Queries SQL no GitHub

SELECT c.segmento,
  SUM(f.valor_total) AS receita,
  COUNT(f.fatura_id) AS n_faturas,
  ROUND(100.0 * SUM(
    CASE WHEN f.estado='Pago'
    THEN f.valor_total ELSE 0 END)
  / SUM(f.valor_total), 1) AS pct_cobrado
FROM Faturas f
JOIN Clientes c USING (cliente_id)
GROUP BY c.segmento;

Como usar os módulos de estudo

Esta área foi criada para transformar estudo em capacidade demonstrável. Em vez de aprender ferramentas de forma solta, cada módulo liga uma competência a problemas reais de negócio: dados financeiros desorganizados, processos manuais, requisitos pouco claros, dashboards pouco úteis e entrevistas onde tens de explicar o teu valor com segurança.

O método é simples: escolhes um módulo, resolves exercícios curtos, guardas a resposta e convertes os melhores exercícios em evidência profissional. No fim, não tens apenas teoria; tens exemplos que podem entrar no GitHub, LinkedIn ou numa conversa de entrevista.

1. Escolhe o foco do diaSeleciona uma área conforme o teu objetivo: SQL/Python para dados, Power BI para dashboards, BA para requisitos, BPMN para processos ou entrevistas para comunicação.
2. Resolve como se fosse trabalho realLê o cenário, identifica a pergunta de negócio, escreve a solução e compara com a resposta sugerida. O importante é treinar raciocínio, não decorar.
3. Guarda prova do progressoQuando um exercício ficar bom, cria uma evidência: query comentada, screenshot, mini case study, user story, mapa BPMN ou explicação para entrevista.
